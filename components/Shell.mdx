This documentation covers the custom React components designed for layout and styling within the [react-print](https://github.com/OnedocLabs/react-print) component library.

## Components Overview

The components provided are intended for use in managing page layout, specifically for documentation purposes. Each component comes with predefined classes for easy styling and is intended to be used as wrappers or standalone layout elements.

### PageTop

Renders a top page wrapper.

```jsx
<PageTop className="your-custom-class">
  {/* Child components */}
</PageTop>
```

**Props**: Accepts all standard HTML div element properties (`React.HTMLProps<HTMLDivElement>`).

### CurrentPageTop

Renders a wrapper indicating the current top of the page, often used for highlighting or marking the current section.

```jsx
<CurrentPageTop className="your-custom-class">
  {/* Child components */}
</CurrentPageTop>
```

**Props**: Accepts all standard HTML div element properties.

### PageBottom

Renders a bottom page wrapper, useful for footer or end-of-page content.

```jsx
<PageBottom className="your-custom-class">
  {/* Child components */}
</PageBottom>
```

**Props**: Accepts all standard HTML div element properties.

### PageBreak

Defines a page break point, ensures all content that follows is displayed on a new page.

```jsx
< PageBreak />
```

**Props**: Accepts all standard HTML div element properties.

### NoBreak

Prevents breaking inside the wrapper, ensuring contained elements stay together on the same page or screen.

```jsx
<NoBreak className="your-custom-class">
  {/* Child components */}
</NoBreak>
```

**Props**: Accepts all standard HTML div element properties.

### FloatBottom

Allows an element to float at the bottom of its container. This is particularly useful for positioning footnotes or similar content.

```jsx
<FloatBottom>
  {/* Child components */}
</FloatBottom>
```

**Props**: Accepts any props (`any`), including children and inline styles.

## Styling

Each component utilizes custom CSS classes prefixed with `--docify-` for easy identification and styling. Additional custom classes can be passed via the `className` prop.

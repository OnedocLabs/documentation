## Prerequisites

To get the most out of this guide, you'll need to:

- [Sign-up/in to Onedoc](https://app.onedoclabs.com/)
- [Generate your API Key](https://app.onedoclabs.com/)

## 1. Installation

Get the Onedoc Node.js SDK

<CodeGroup>

```sh npm
npm install -s @onedoc/client
```

```sh yarn
yarn add @onedoc/client
```

```sh pnpm
pnpm add @onedoc/client
```

</CodeGroup>

## 2. Create a document template

Start by creating your document template on `components/pdf-template.js`.<br/>Your file must export a React component.

```javascript components/pdf-template.js
import * as React from "react";

export const PDFTemplate = ({ name }) => {
  return (
    <div>
      <h1> Hello {name} !</h1>
    </div>
  );
};
```

and optionally, a stylesheet:

```css stylesheet.js
body {
  background-color: lightblue;
}
```
## 3. Generate your PDF

Get your API key at https://app.onedoclabs.com/.

Import the React document template and generate a PDF using your `ONEDOC_API_KEY` and your main React component.

<CodeGroup>

```javascript generate.js
import { PDFTemplate } from "./components/pdf-template";
import { Onedoc } from '@onedoc/client"';
import { readFileSync, writeFileSync } from "fs";
import { renderToString } from "react-dom/server";

const onedoc = new Onedoc("ebec0c4c-214f-4796-afd2-b5c9b12281b6");

let doc = {
  html: renderToString(<PDFTemplate name="Bruce Wayne" />),
  title: "Hello",
  assets: [
    {
      path: "./stylesheet.css",
      content: readFileSync(path.join(process.cwd(), "stylesheet.css"))
        .toString(),
    },
  ],
};

const { file, info, error } = await onedoc.render(doc, dev = false); //dev = false renders the document with a watermark);

if (error) {
  throw error;
}

writeFileSync(",/hello.pdf", Buffer.from(file));
```

</CodeGroup>

Stylesheets and other referenced assets should be passed in the `asset` parameter. If you are using a framework such as Tailwind, you must submit a <b>compiled stylesheet</b> that can be applied to a HTML file that contains the compiled React component. <br/>
You must upload the content of your asset in string or Buffer format in the `content` field.

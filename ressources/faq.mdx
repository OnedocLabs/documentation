<AccordionGroup>
  <Accordion title="What is Onedoc ?">
    OneDoc is an API solution that allows developers to create, manage, and automate PDF document generation using familiar technologies like React. It simplifies the process of document creation, analysis and tracking. 
  </Accordion>
  <Accordion title="Is OneDoc suitable for developers with no prior experience in React?">
    Of course. While familiarity with React is beneficial, OneDoc is designed to be user-friendly. Moreover, OneDoc offers [SDKs](./sdks) covering many programming languages and is constantly evolving to provide the best user experience.
  </Accordion>
  <Accordion title="How secure is OneDoc when handling sensitive documents?">
    OneDoc prioritizes security, employing advanced encryption for data in transit and at rest, along with stringent authentication protocols to ensure that sensitive documents are protected at all times. [More about security](./security).
  </Accordion>
  <Accordion title="Can OneDoc be integrated with other software or platforms?">
    Yes, OneDoc is built for compatibility and can be seamlessly integrated with various platforms and systems, enhancing its functionality and user convenience. More about [integrations](./integrations)
  </Accordion>
  <Accordion title="Is there an open-source component to OneDoc?">
  OneDoc embraces the open-source community by making portions of its code available. This allows developers to contribute, review, and enhance the software, fostering innovation and reliability. Check out [github](www.github.com)
  </Accordion>
  <Accordion title="What kind of support does OneDoc offer for troubleshooting and queries?">
    OneDoc provides robust support through email support, online call for entreprise, and a community forum where users can find assistance for troubleshooting and general queries. [Join us!](communitylink)
  </Accordion>
  <Accordion title="Are there any templates available, or do users need to design PDFs from scratch?">
    OneDoc offers a range of pre-designed templates to get users started. However, for specific needs, users can also create custom templates from scratch, providing greater flexibility and control. We are currently establishing a template marketplace where the community can share their creations with others. [More about our templates](/templates/introduction)
  </Accordion>
  <Accordion title="Does OneDoc handle different document formats and layouts?">
    OneDoc is versatile in handling various document formats and layouts, allowing users to create a wide range of document types, from simple text documents to complex reports with diverse layouts.
  </Accordion>
